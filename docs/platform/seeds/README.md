# Platform Seed Documents

These seed docs are normalized reference inputs for WAR API guidance.

## Purpose

- Hold reusable, reviewable knowledge fragments used by docs tooling and MCP.
- Avoid embedding large API catalogs in instruction files.
- Keep confidence and uncertainty explicit.

## Source Priority

1. Generated canonical docs under `docs/war-api/`
2. MCP server responses built from canonical docs
3. Seed docs in this folder (curated summaries, conventions, known caveats)

## Maintenance Rules

- Add focused facts, not full dumps.
- Include confidence notes for inferred behavior.
- Update seeds when implementation/research improves certainty.
